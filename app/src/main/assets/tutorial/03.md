## 模型配置

SolveX 需要配置 AI 模型才能正常解析内容。在「设置」中可以管理你的服务提供商和助手。


### 添加服务提供商

服务提供商就是 AI 模型的来源。SolveX 兼容 OpenAI 接口协议，市面上绝大多数 AI 服务商都可以使用。

**配置步骤**：

1. 填写 API 地址（不用加 `/v1` 后缀，会自动拼接）
2. 输入 API 密钥（注意区分大小写，多余空格会被自动去除）
3. 点击「获取模型列表」拉取可用模型
4. 选择默认使用的模型
5. 点击「接口预览」可以测试连接是否正常

> 一个提供商可以配置多个模型，系统会根据当前引擎自动选择对应的模型。


### 管理智能助手

助手是 AI 的「角色设定」，同一个模型可以创建多个不同用途的助手。

**创建与编辑**：进入「设置 → 助手管理」即可创建新助手或编辑现有助手。

**调整顺序**：支持长按拖拽排序，排在前面的助手会在首页优先显示。

**常用助手示例**：

- **题目解答助手**：分步书写解题过程，标注公式，验算易错点
- **聊天总结助手**：提取聊天记录的核心话题、主要观点和待办事项

> 切换助手无需重启应用，下次解析自动生效。


### 参数调整

**首字超时时间**

从发送请求到 AI 返回第一个字符的最长等待时间：

- 国内开源模型：默认 10 秒即可
- 境外中转模型：建议设为 30~60 秒

**自动滚动内容**

开启后，解析结果和历史记录详情会自动滚动到最新内容，省去手动滑动的麻烦。


### 常见排查

**模型无法识别或报错**

1. 检查 API 地址是否正确（不要带 `/v1`）
2. 检查 API 密钥是否有效
3. 在提供商页面点击「获取模型列表」验证连接
4. 切换引擎试试（长按悬浮球）

**检查更新提示网络异常**

1. 检查设备网络连接
2. 尝试切换 WiFi 或移动数据

**服务异常**

若遇到服务崩溃或异常，SolveX 内置了全局异常捕获机制，会自动记录错误日志。如果问题持续出现，可以重启应用或联系开发者反馈。
